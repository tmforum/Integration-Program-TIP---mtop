Class AlarmNotification

This object class represents the information in an alarm notification.
Traceability: R_TMF518_NRA_I_0017.

Attributes
Boolean  isEdgePointRelated

This attribute indicates whether the event is related to a Termination Point (TP) at the edge of a subnetwork.
It is True for edge points and their contained CTPs although it is always False as a CTP's attribute if the CTP doesn't terminate a Topological Link.
Note that the value is True for all notifications reporting a value change of the TP's attribute 'isEdgePoint' even if the new value is False.
Boolean  isClearable

This attribute is True if the alarm/event represents a condition that will not be cleared at a later time (or is itself a clear); False otherwise.
Traceability: R_TMF518_NRA_I_0019.
 layerRate

This attribute identifies the layer rate to which the Alarm is relevant to.
String  probableCause

This attribute identifies further qualification as to the Probable Cause of the alarm. No other string than the ones defined in the ProbableCause class shall be used for this field. Refer to the attached SD1-33_ProbableCauses supporting document for the currently specified probable cause names.
Traceability: R_TMF518_NRA_I_0020.
String  probableCauseQualifier

This attribute may further qualify the source of the Alarm.
Traceability: R_TMF518_NRA_I_0018.
 perceivedSeverity

This attribute identifies the severity of the Alarm.
Traceability: R_TMF518_NRA_I_0021.
ServiceAffecting  serviceAffecting

This attribute identifies whether the Alarm has affected service.
Traceability: R_TMF518_NRA_I_0022.
 affectedPtpRefList

This attribute identifies all affected associated PTP names.
String  additionalText

This attribute contains a free form text description to be reported. The requested OS is not required to understand the semantics of this field for interpretation of the notification.
Traceability: R_TMF518_NRA_I_0024.
 aliasNameList

This attribute contains a list of zero or more alias names for the object who raise the Alarm.
String  nativeProbableCause

This attribute identifies the Probable Cause as portrayed on the target OS user interface.
AcknowledgeIndication  acknowledgeIndication

This attribute identifies if the alarm has been acknowledged or not. In case the EMS does not support acknowledgement for this event or does not support acknowledgement at all NOT_APPLICABLE shall be used.
Traceability: R_TMF518_NRA_I_0025.
Boolean  rootCauseAlarmIndication

This attribute indicates whether an alarm is a raw (uncorrelated) alarm or a root cause alarm indication.
The attribute can have two values:
- True (meaning that the alarm is a Root Cause Alarm Indication)
- False (meaning the alarm is a raw alarm).
String  x733_EventType

This attribute, when present in an alarm notification, classifies the alarm into one of the five basic categories specified in ITU-T X.733.
This is consistent with the ITU-T X.733 definition.

Value is one of the following:
- "communicationsAlarm",
- "environmentalAlarm",
- "equipmentAlarm",
- "processingErrorAlarm",
- "qualityOfServiceAlarm".
String  x733_SpecificProblems

This attribute, when present in an alarm notification, identifies further refinements to the Probable Cause of the alarm.
Note: It is similar to ProbableCauseQualifier, but this parameter is designed to be human readable and compatible with ITU usage.
This is consistent with the ITU-T X.733 definition.
String  x733_BackedUpStatus

This attribute, when present in an alarm notification, specifies whether or not the object emitting the alarm has been backed-up, and services provided to the user have, therefore, not been disrupted.
This is consistent with the ITU-T X.733 definition.

Value is one of the following:
- "BACKED_UP"
- "NOT_BACKED_UP".
 x733_BackUpObject

This attribute, when present in an alarm notification, specifies the object that is providing back-up services for the object about which the notification pertains.
This parameter shall be present when the Backed-up status parameter is present and has the value "BACKED_UP".
This is consistent with the ITU-T X.733 definition.
String  x733_TrendIndication

This attribute, when present in an alarm notification, specifies the current severity trend of the object. If present it indicates that there are one or more alarms ("outstanding alarms") which have not been cleared, and pertain to the same object as that to which this alarm ("current alarm") pertains.
This is consistent with the ITU-T X.733 definition.

Value is one of the following:
- "MORE_SEVERE",
- "NO_CHANGE",
- "LESS_SEVERE".
CorrelatedNotifications  x733_CorrelatedNotificationList

This attribute, when present in an alarm notification, contains a set of Notification identifiers and, if necessary, their associated object names.
This set is defined to be the set of all notifications to which this notification is considered to be correlated.
The source object name shall be present if the correlated event report is from an object other than the one in which the Correlated Notifications parameter appears. Otherwise, it is Null (empty list).
This is consistent with the ITU-T X.733 definition.
 x733_MonitoredAttributeList

This attribute, when present in an alarm notification, defines one or more attributes of the managed object and their corresponding values at the time of the alarm.
This is consistent with the ITU-T X.733 definition.
String  x733_ProposedRepairActionList

This attribute, when present in an alarm notification, is used if the cause is known and the system being managed can suggest one or more solutions (such as switch in standby equipment, retry, replace media).
This is consistent with the ITU-T X.733 definition.
 x733_AdditionalInformation

This attribute, when present in an alarm notification, allows the inclusion of a set of additional information in an alarm notification.
This is consistent with the ITU-T X.733 definition.
Note: The same information can be directly encoded as separate parameters of the notification. However, this parameter is retained for consistency with ITU-T X.733.
