Interface Common

This interface contains the operations common to all Interfaces.

Operations
 getVersion

This operation returns the version of the interface that is supported.
This operation should be called by a requesting OS before other communications. The requesting OS can determine from the response string which version of the interface is available.

 getCapabilities 
Public  getCapabilities(AttributeValuePair )

The operation returns the capabilities of the target OS with information whether they are supported or not supported by the target OS (including the connection management modes indicating whether CC sharing, pending SNCs, and termination inclusion are supported; see attached supporting document modesOfOperation for explanation).
Traceability: R_TMF518_FMW_II_0022.

Parameters:
AttributeValuePair capabilities [*]This parameter contains the list of capabilities with information whether they are supported or not supported.
The currently defined values are as follows:
(Attribute Name - Value Pair List) (Capability Name - Supported/Unsupported Value Pair List).

For Connectionless networks, the "Connectivity_Awareness" capability is defined as follows:
(Attribute Name - Value Pair List) ("Connectivity_Awareness" - ConnectivityAware/ConnectivityUnaware)
- "connectivity aware" indicates that the target OS has some capability to detect the connectivity between the MFDs within an FD;
- "connectivity-unaware" indicates that the target OS has no capability at all to detect connectivity between the MFDs within an FD.
