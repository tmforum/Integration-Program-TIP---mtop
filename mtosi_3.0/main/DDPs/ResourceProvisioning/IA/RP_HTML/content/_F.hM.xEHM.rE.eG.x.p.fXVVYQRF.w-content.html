Type Definitions
Data Type CallCreateData

This data type represents the parameters that are required to create a Call. It will be passed as one of the parameters in the establishCall operation.

Attributes
 aEnd

This attribute defines the A-end of the Call.

It may be one of the following fields:
- SNPP (Control Plane name space) which may be resolved to the SNP level
- SNP (Control Plane name space)
- CPTP/FTP/CTP/PTP (Transport Plane name space)
- TNA Name (which identifies either a TNA or a Group-TNA Name in the Control Plane name space)
- Null.

During Call Creation the A-end and Z-end may be of different types (e.g. SNPP name at A-end and TNAName at Z-end).
This attribute can be null when the specific details of the end points are supplied in the connection specifications.
For a detailed description of the use of this attribute see SD1-45_ASONControlPlaneManagement-Primer.

Note: In a Control Plane environment, the type of the A-end is independent of the type of the Z-end.
 zEnd

This attribute defines the Z-end of the Call. See also aEnd description.
 callParameters

This attribute provides the call parameters:
- Severely Degraded Threshold
- Degraded Threshold
- Class Of Service
- Class Of Service specific Parameters.
Note: The parameters "Class of Service" and "Class of Service specific Parameters" are only relevant to an established Call and shall be ignored during Call creation in a VCAT environment.
 callDiversity

This attribute defines the diversity and co-routing requirements related to the Call.
