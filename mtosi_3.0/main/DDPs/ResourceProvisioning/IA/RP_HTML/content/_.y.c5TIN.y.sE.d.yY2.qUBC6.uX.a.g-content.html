Type Definitions
Data Type RouteCreateData

This data type represents the parameters that may be provided when adding a Route to a Subnetwork Connection.

Attributes
Boolean  isIntendedRoute

This attribute defines the purpose of the Route to be created. Refer to description in the Route object class.
Boolean  isExclusiveRoute

This attribute defines that not any other Routes of other Subnetwork Connections can share any of the Routes Cross Connections or Connection Termination Points, even in pending state (true).
Boolean  isFullRoute

This attribute indicates whether the provided inclusion constraints specifies the complete Route details (as opposed to only a partial constraint). When no inclusions constraints are specified, false must be used.
 additionalInfo

This attribute allows the communication from the requesting OS to the target OS of additional information which is not explicitly modelled.
 ccInclusionList

This attribute specifies a list of Cross-Connects that must be used by the Route.
 inclusionResourceRefList

This attribute specifies a list of Termination Points, Managed Elements, Topological Links, Group Termination Points and server Subnetwork Connections that must be used by the Route.
The list must be empty if no inclusion constraints are required.
If the target OS cannot fully satisfy the routing constraints provided, then the request will be rejected.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if exclusionResourceRefList is not empty.
 exclusionResourceRefList

This attribute specifies a list of Termination Points, Subnetwork Connections, Managed Elements, Topological Links and Group Termination Points that must not be used by the Route.
The list must be empty if no exclusion constraints are required.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if inclusionResourceRefList or ccInclusionList is not empty.
