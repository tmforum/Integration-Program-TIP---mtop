 createAndActivateFlowDomainFragment

In case not all requested (edge) Flow Points can be connected when the FDFr is created, the target OS returns the list of the not connectable Connectionless Port Termination Points (CPTPs) and Flow Points in the success reply.
If internal CPTPs are provided by the requesting OS and if it is not possible to include all provided internal CPTPs to the route of the FDFr, the operation will be rejected.

 deleteMatrixFlowDomain

Traceability: R_TMF518_RP_II_0041, UC_TMF518_RP_0023.
 modifyFlowDomain

This operation modifies a Flow Domain (FD). The modification of the Flow Domain is done on a best effort basis. All attributes that could not be modified shall be returned in the reply.

Traceability: R_TMF518_RP_II_0048, R_TMF518_RP_II_0049, UC_TMF518_RP_0017.
 modifyFlowDomainFragment

This operation modifies a Flow Domain Fragment (FDFr).
The operation is able to modify
- the attributes and parameters of the FDFr
- the parameters of other involved Connectionless Port Termination Points (CPTPs) and Flow Points (FPs)
- the FDFr by addition/removal of FPs.
FPs are added by designating the FD Edge CPTPs and FD Internal CPTPs to be added in two separate lists (one for edge and one for internal). FPs are removed by designating their server CPTPs (edge and internal) in a third list. The removal of CPTPs shall be rejected if less than two end Flow Points would remain after successful completion of the operation.

Traceability: R_TMF518_RP_II_0062, R_TMF518_RP_II_0063, R_TMF518_RP_II_0065, R_TMF518_RP_II_0066, UC_TMF518_RP_0033.
 modifyMatrixFlowDomain

This operation modifies a Matrix Flow Domain (MFD). The modification of the Matrix Flow Domain is done on a best effort basis. All attributes that could not be modified shall be returned in the reply.

Traceability: R_TMF518_RP_II_0039, R_TMF518_RP_II_0040, UC_TMF518_RP_0024.
 unassignConnectionlessPortTerminationPointsFromMfd

This operation un-assigns Connectionless Port Terminations Points (CPTPs) from the Matrix Flow Domain (MFD). The provided CPTPs must be potential CPTPs for this MFD (e.g. have to be on the same equipment or same rack with backplane connectivity).

 modifyFlowDomainFragment 
Public  modifyFlowDomainFragment(String, , , , FlowDomainFragmentModifyData, ,  )

This operation modifies a Flow Domain Fragment (FDFr).
The operation is able to modify
- the attributes and parameters of the FDFr
- the parameters of other involved Connectionless Port Termination Points (CPTPs) and Flow Points (FPs)
- the FDFr by addition/removal of FPs.
FPs are added by designating the FD Edge CPTPs and FD Internal CPTPs to be added in two separate lists (one for edge and one for internal). FPs are removed by designating their server CPTPs (edge and internal) in a third list. The removal of CPTPs shall be rejected if less than two end Flow Points would remain after successful completion of the operation.

Traceability: R_TMF518_RP_II_0062, R_TMF518_RP_II_0063, R_TMF518_RP_II_0065, R_TMF518_RP_II_0066, UC_TMF518_RP_0033.

Parameters:
String errorReason [1]This parameter returns the error(s) if any.
failedTpRefList [*]This parameter returns the list of Connectionless Port Termination Points or Flow Points (requested in the modifyData parameter) that could not be added or could not be removed, whichever was requested.
fdfrRef [1]This parameter provides the name of the Flow Domain Fragment to be modified.
modifiedFdfr [1]This parameter returns the modified Flow Domain Fragment object.
FlowDomainFragmentModifyData modifyData [1]This parameter provides the parameters that shall be modified in the Flow Domain Fragment.
