Type Definitions
Data Type ConnectionCreateData

This data type represents the parameters that are required to create a Connection.

Attributes
 direction

This attribute defines the directionality of the Connection (bidirectional or unidirectional).
 staticProtectionLevel

This attribute defines the level of internal resilience/protection of the Connection as specified in the attached supporting document SNCTypes.
ProtectionEffort  protectionEffort

This attribute defines the requested level of effort regarding the staticProtectionLevel.
 rerouteAllowed

This attribute defines if the target OS/Control Plane is allowed and/or required to reroute the Connection if there is a failure on the Connection, periodically to optimize the routes, or for any other reason. It is an target Os/Control Plane implementation whether this is done using network routing protocols or if the target OS/Control Plane detects the failure and takes appropriate action to attempt to fix the Connection.

The NO value means that the target OS/Control Plane is not allowed to reroute the Connection.
The YES value means that the target OS/Control Plane is allowed to reroute the Connection and is required to attempt to reroute it upon failure.
The NA value is used when the requesting OS does not want to specify the exact target OS/Control Plane behaviour. In this case it is left up to the target OS/Control Plane to decide whether rerouting will be provided. It is also used if the target OS/Control Plane is allowed to reroute the Connection but not required to attempt to reroute it upon failure.
 connectionType

This attribute defines the type of the Connection. The Connection types are specified in the attached supporting document SNCTypes.

Notes:
For the top level Connection only simple will be used in this version.
Add-drop-A and Add-drop-Z are only for subordinate Connections in this version.
 layerRate

This attribute defines the layer rate of the Connection.
 ccInclusionList

This attribute specifies a list of cross-connects that must be used by the Control Plane Connection.
These are Routing Node connections, ended by SNPs or SNPPs, and the optional TNA/FTP/CTP end point values are provided by a/zEndTpList, a/zEndTnaNameOrGroupTnaName parameters.
The list must be empty if no cross-connect constraints are required.
 inclusionResourceRefList

This attribute specifies a list of Termination Points, Managed Elements, Topological Links, Group Termination Points, Multi Layer Routing Areas, Multi Layer Subnetwork Point Pools and Multi Layer Subnetwork Point Pool Links that must be used by the Connection.
The list must be empty if no inclusion constraints are required.
If the target OS cannot fully satisfy the routing constraints provided, then the request will be rejected.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if exclusionResourceRefList is not empty.
 exclusionResourceRefList

This attribute specifies a list of Termination Points, Managed Elements, Topological Links, Group Termination Points, Multi Layer Routing Areas, Multi Layer Subnetwork Point Pools, Multi Layer Subnetwork Point Pool Links and Subnetwork Connections that must not be used by the Connection.
The list must be empty if no exclusion constraints are required.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if inclusionResourceRefList is not empty.
 aEnd

This attribute defines the source end point of the Connection. The value of this parameter is in one of the following name types:
- SNPP (Control Plane name space)
- SNP (Control Plane name space)
- TP (Transport Plane name space)
- TNA Name (which identifies either a TNA or a Group-TNA Name in the Control Plane name space).
 zEnd

This attribute defines the sink end point of the Connection. The value of this parameter is in one of the following name types:
- SNPP (Control Plane name space)
- SNP (Control Plane name space)
- TP (Transport Plane name space)
- TNA Name (which identifies either a TNA or a Group-TNA Name in the Control Plane name space).
Boolean  isReportingAlarms

This attribute indicates whether alarm reporting for the Connection is administratively activated or de-activated.
True = alarm reporting is activated; false = alarm reporting is de-activated.
Boolean  isRevertive

This attribute defines whether the Connection shall always attempt to return to it's home (intended) Route.
Integer  priority

This attribute indicates the requested priority of the Connection to be created.
Allowed values are = "0", "1", "2", ... where zero (0) is the highest priority.
 asapRef

This attribute indicates the assignment of an Alarm Severity Assignement Profile (ASAP) to the Connection.
 routingConstraintEffort

This attribute indicates the requested level of effort to meet the routing constraint when creating the Connection.
 maximumCost

This attribute defines a number that indicates the maximum cost allowed for a Connection. The cost of a Connection is calculated based on the cost of the supporting MLSNPP links. This link cost represents a vector of one or more metrics, each of which indicates the relative desirability of a particular link over another during path selection.
String  routeGroupLabel

This attribute represents the Route Group the Connection is associated to. When a target OS creates a Call with diverse Route Groups, it assigns a Route Group label (which may be a number) to each Connection.
Connections in the same Route Group will be assigned the same value for routeGroupLabel. Connections in different Route Groups will be assigned different values for routeGroupLabel.
 connectionParameterProfileRef

This relationship defines the set of class of service related parameters that are associated with the Connection.
