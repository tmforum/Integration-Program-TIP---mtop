Type Definitions
Data Type EquipmentModifyData

This data type represents the specific parameters that may be provided when modifying an Equipment.

Attributes
String  expectedEquipmentObjectType

This attribute defines the type of the expected Equipment. Values are vendor-specific.
Boolean  isReportingAlarms

This attribute indicates whether alarm reporting for the Equipment is administratively activated or de-activated.
True = alarm reporting is activated; False = alarm reporting is de-activated.
String  manufacturer

This attribute identifies the required Equipment manufacturer name. It is defined as a non-empty free format string with no semantics.
In cases where the requesting OS provisions Equipment with a particular manufacturer name and this name conflicts with the manufacturer name as discovered by the target OS, the manufacturer name discovered by the target OS shall be considered the correct name, and the target OS shall issue an AVC notification to the requesting OS indicating a change to the manufacturer name.
 protectionRole

This attribute defines the protection role that the Equipment plays in case it takes part in an Equipment protection scheme. In case the Equipment is not protected, the value NOT_APPLICABLE shall be used.
 protectionSchemeState

This attribute identifies the individual lock of this Equipment when it is participating in an Equipment protection scheme. In case the Equipment is not protected, the value "UNKNOWN" shall be used.
 asapRef

This attribute indicates the new assignment of an AlarmSeverityAssignementProfile (ASAP) to the Equipment.
