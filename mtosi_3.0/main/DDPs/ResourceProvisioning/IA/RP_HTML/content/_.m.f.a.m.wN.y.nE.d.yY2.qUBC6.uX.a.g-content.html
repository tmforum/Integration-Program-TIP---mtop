 launchGuiCutThrough 
Public  launchGuiCutThrough(, , String, String, Boolean, ,  )

This operation of the EMS invokes the launch of the GCT under a server-launched configuration.
See R_TMF518_RP_II_0067, R_TMF518_RP_II_0070, UC_TMF518_RP_0036.

Parameters:
additionalInfoInputList [*]This parameter provides additional information provided to the EMS. The content of this parameter is subject to bilateral agreements.
additionalInfoOutputList [*]This parameter returns additional information provided by the EMS. The content of this parameter is subject to bilateral agreements.
String context [1]This parameter provides the window context. Context is of secondary priority with the Object Name as serving as the highest priority in determining which GCT window to open.
Suggested window contexts are:
- TOP_LEVEL,
- FAULT,
- CONFIGURATION_SOFTWARE,
- CONFIGURATION_CONNECTION,
- ACCOUNTING,
- PERFORMANCE,
- SECURITY,
- SYSTEMS_MANAGEMENT.
String displayAddress [1]This parameter provides the IP address or hostname including any display port information.
Boolean isClosingEnabled [1]This parameter returns a flag indicating the ability of the EMS to manage the open GCT window. Specifically whether the EMS is able to close it upon request (i.e. using a subsequent destroyGCT operation).
objectName [1]This parameter provides the applicable reference object. The EMS displays the GCT to the extent supported by the EMS. The EMS should invoke the GCTwhich is available for the closest object in the object name hierarchy. (e.g., if the card internal view is not available for the equipment object indicated then the EMS will invoke the GCT of its Managed Element instead). Only the top-level of the EMS is mandatory to be supported by the EMS.
userInfoList [*]This parameter provides the capability / user information that applies to the operations that are made available through the GCT application, depending on the EMS ability to enforce capability / user limitations on the GCT.
Suggested information is:
- userId,
- password,
- capability.
Capability suggested values are:
- READ_ONLY,
- READ_WRITE.
Values and meaning for these attributes are left to bilateral agreement EMS and NMS.
Returns:
void [1]
Constraints:
<Unnamed Constraint> = "1. The EMS supports the server launch window system protocol (i.e., is capable of server launching in addition to client launching). 2. The NMS has retrieved the GCT profile of the EMS that contains the window/widget data the EMS supports for GCT, and has selected an object to be displayed upon GCT launch. 3. The managed object to be displayed upon GCT launch exists and is in a valid state. 4. The EMS has access to the NMS display address. 5. The NMS has access to any other required servers (e.g., ICA server for Citrix Metaframe). "
<Unnamed Constraint> = "The GCT application opens the display successfully at the NMS host address."
