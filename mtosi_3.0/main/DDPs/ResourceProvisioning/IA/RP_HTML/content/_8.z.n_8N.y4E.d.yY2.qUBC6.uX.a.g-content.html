Type Definitions
Data Type GuiCutThroughProfileInfo

This data type provides EMS profiling information that applies to the GUI Cut-through (GCT):
- indication of support for the "server-launch",
- the supported GCT contexts and scopes,
- the GCT command information necessary to launch the GCT on the client side.

Attributes
ServerLaunchCapability  serverLaunchCapability

This attribute indicates whether the EMS supports the launchGCT operation to launch the GCT on the server-side. If this value is CLIENT_LAUNCH_ONLY the launch can only be carried out outside the NMS-EMS interface (i.e. a "client-launch").
String  gctHostName

This attribute contains the IP address (or host name which is mapped to the appropriate IP address) that allows the NMS to identify where to launch the EMS GUI client. This string can be empty only in case the attribute platform is "local" or "web-based".
For Citrix commands, it is the TSE (Terminal Server Environment) IP address.
Platform  platform

This attribute indicates the platform supported for the GUI Cut-Through. Valid values are: "unix", "windows", "local", and "web-based". This string shall not be empty except for local or web-based GUI.
GuiCutThroughData  gctDataList

This attribute defines the list of records associated with each supported window type and how to launch them. If the EMS does not provide data for a desired window type, the NMS may use an alternative data record such as the top level window. (The EMS top level window is the only window that is mandatory for the EMS to support.)
