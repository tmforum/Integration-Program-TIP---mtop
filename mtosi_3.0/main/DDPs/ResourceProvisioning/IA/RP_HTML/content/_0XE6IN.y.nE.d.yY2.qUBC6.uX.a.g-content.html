 verifyTransmissionDescriptorAssignment

The operation verifies the Transmission Descriptor (TMD) state of the specified resource object identified by the resource reference.
According to the requested transmission direction it checks whether a TMD is assigned as egress and/or ingress or neutral TMD to the resource. If this is true it first verifies the existence of the assigned TMD(s) and then compares the Transmission Parameters of the TMD with the Transmission Parameters of the resource. After this the additionalObjectInfo parameters of the TMD are compared with the additionalInfo parameters of the resource. If a TMD Transmission Parameter is not present as a resource Transmission Parameter or is present but has a different value, this is called a "parameter mismatch", and the TMD state will take the value "mismatch". The operation returns the current TMD state and all mismatched Transmission or additionalObjectInfo Parameters of the TMD.
Traceability: R_TMF518_RP_II_0099.

 verifyTransmissionDescriptorAssignment 
Public  verifyTransmissionDescriptorAssignment(, , , ,  )

The operation verifies the Transmission Descriptor (TMD) state of the specified resource object identified by the resource reference.
According to the requested transmission direction it checks whether a TMD is assigned as egress and/or ingress or neutral TMD to the resource. If this is true it first verifies the existence of the assigned TMD(s) and then compares the Transmission Parameters of the TMD with the Transmission Parameters of the resource. After this the additionalObjectInfo parameters of the TMD are compared with the additionalInfo parameters of the resource. If a TMD Transmission Parameter is not present as a resource Transmission Parameter or is present but has a different value, this is called a "parameter mismatch", and the TMD state will take the value "mismatch". The operation returns the current TMD state and all mismatched Transmission or additionalObjectInfo Parameters of the TMD.
Traceability: R_TMF518_RP_II_0099.

Parameters:
additionalObjectInfo [*]This parameter returns a "delta" list of mismatched additionalObjectInfo parameters, or is empty.
direction [1]This parameter identifies the direction for which the Transmission Descriptor assignment(s) of the resource shall be verified.
SOURCE refers to the egress TMD state.
SINK refers to the ingress TMD state.
BIDRECTIONAL refers to the egress and ingress or neutral TMD states.
UNIDIRECTIONAL is not allowed.
resourceRef [1]This parameter provides the name of the resource whose TMD assignment shall be verified.
tmdAssignmentState [1]This parameter returns the TMD state of the resource as determined by the verification procedure.
If both the egress and ingress TMD states are requested and their values are different, the concatenation of the egress TMD state with the ingress TMD state is returned, separated by a semicolon.
transmissionParameterList [*]This parameter returns a "delta" list of mismatched TMD Transmission Parameters, or is empty.
See attached supporting document SD1-16_LayeredParameters for more details.
Returns:
void [1]

<Unnamed Constraint> = "None"
<Unnamed Constraint> = "No change in the system."
