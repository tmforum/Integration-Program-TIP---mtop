Interface RemoteUnitControlService

Operations
 createRemoteUnit

This operation allows the requesting OS to create a remote unit (e.g., DSL NT, GPON ONT). The successful result of this operation is the creation of the EquipmentHolder(s), the Equipment(s) and all its related objects such as termination points.

See attached supporting document SD2-20_EquipmentModel.

Traceability: R_GPON_I_0002, R_GPON_II_0013, UC_GPON_0001.
 deleteRemoteUnit

This operation allows the requesting OS to delete a Remote Unit (e.g., DSL NT, GPON ONT). The operation will delete the corresponding Equipment object(s), all containing EquipmentHolder(s) and all its related objects such as termination points of the remote unit.

Traceability: R_GPON_I_0002, R_GPON_II_0014, UC_GPON_0002.
 modifyRemoteUnit

This operation allows the requesting OS to modify a Remote Unit (e.g., DSL NT, GPON ONT). The operation allows one to modify the corresponding Equipment object(s), all containing EquipmentHolder(s) and all its related objects such as termination points of the remote unit.

Traceability: R_GPON_I_0002, R_GPON_II_0015, UC_GPON_0003.

 createRemoteUnit 
Public  createRemoteUnit(RemoteUnitCreateData, , , , String, , , String )

This operation allows the requesting OS to create a remote unit (e.g., DSL NT, GPON ONT). The successful result of this operation is the creation of the EquipmentHolder(s), the Equipment(s) and all its related objects such as termination points.

See attached supporting document SD2-20_EquipmentModel.

Traceability: R_GPON_I_0002, R_GPON_II_0013, UC_GPON_0001.

Parameters:
RemoteUnitCreateData createData [1]This parameter provides the data that describes the remote unit to create.
createdEquipmentHolderList [*]This parameter returns the resulting created EquipmentHolder(s).
createdEquipmentList [*]This parameter returns the resulting created Equipment(s).
createdPtpList [*]This parameter returns the resulting created PhysicalTerminationPoint(s).
String errorReason [0..1]This parameter returns the activation error(s), if any.
managedElementRef [1]This parameter provides the name of the ManagedElement (e.g., DSLAM, GPON OLT) that will contain the remote unit.
profileRefList [*]This parameter provides the list of the non-layer oriented profiles to configure the new remote unit.

Traceability: R_GPON_II_0012
String remoteUnitId [1]This parameter provides the identifier for the remote unit.
The value of this parameter will be placed into "" in the [/remote_unit=] part of the new top most EquipmentHolder name attribute.
Returns:
void [1]
Constraints:
<Unnamed Constraint> = "None"
<Unnamed Constraint> = "1. The new Equipment object is created and depending on the target OS, any object supported by the Equipment. 2. The Equipment Holder State reflects that the Equipment is now EXPECTED_AND_NOT_INSTALLED."
