Type Definitions
Data Type FlowDomainFragmentSetData

This data type contains the parameters that can be passed when a Flow Domain Fargement is created or modified.

Traceability: R_TMF518_RP_II_0061, R_TMF518_RP_II_0063.

This data type contains the parameters that can be passed when a Flow Domain Fargement is created or modified.

Traceability: R_TMF518_RP_II_0061, R_TMF518_RP_II_0063.

Attributes
ConnectivityRequirement  connectivityRequirement

This attribute identifies for a "connectivity-aware" target OS the requested operation mode in case not all Flow Points have potential connectivity to one another.
If the target OS is not connectivity-aware, this parameter is ignored.
 administrativeState

This attribute indicates whether the Flow Domain Fragment (FDFr) shall be locked (i.e., traffic units cannot flow through the FDFr) or unlocked (i.e., traffic units are allowed to flow through the FDFr).
 aEndEdgeTpRefList

This attribute identifies the list of Connection Port Termination Point (CPTP) names that delimit the Flow Domain Fragment (FDFr) and characterize its edges (entrance and/or exit points). As a result of creating/modifying the FDFr, Flow Points are created as clients of the FD Edge CPTPs.
In case of a unidirectional FDFr this attribute contains the list of source FD Edge CPTPs. In case of a bidirectional FDFr this attribute may be combined with the zEndEdgeTPs relationship to obtain all the FD Edge CPTPs that have to be associated to the FDFr.
 internalTpRefList

This attribute identifies the list of internal Connection Port Termination Point (CPTP) names that must be included in / or added to the route of the Flow Domain Fragment (FDFr). As a result of creating/modifying the FDFr, Flow Points are created as clients of the internal CPTPs.
 zEndEdgeTpRefList

This attribute identifies the list of Connection Port Termination Point (CPTP) names that delimit the Flow Domain Fragment (FDFr) and characterize its edges (exit points). As a result of creating/modifying the FDFr, Flow Points are created as clients of the FD Edge CPTPs.
In case of a unidirectional FDFr this attribute contains the list of sink FD Edge CPTPs. In case of a bidirectional FDFr this attribute may be combined with the aEndEdgeTPs relationship to obtain all the FD Edge CPTPs that have to be associated to the FDFr.
 transmissionParameterList

This attribute contains the layer rate on which connectivity is established and a related set of connectionless parameters.
Refer to chapter "Connectionless Technology Parameters" of the attached supporting document SD1-16_LayeredParameters for details of the currently defined connectionless parameters.
