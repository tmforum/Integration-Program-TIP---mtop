Type Definitions
Data Type EquipmentHolderSetData

This data type represents the specific parameters that may be provided when creating/modifying an EquipmentHolder.

Attributes
 asapRef

This attribute indicates the required assignment of an Alarm Severity Assignment Profile (ASAP) to the EquipmentHolder.
Boolean  isReportingAlarms

This attribute indicates whether alarm reporting for the EquipmentHolder is administratively activated or de-activated.
True = alarm reporting is activated; False = alarm reporting is de-activated.
String  manufacturer

This attribute identifies the required Equipment Holder manufacturer name. It is defined as a non-empty free format string with no semantics.
In cases where the requesting OS provisions the EquipmentHolder with a particular manufacturer name and this name conflicts with the manufacturer name as discovered by the target OS, the manufacturer name discovered by the target OS shall be considered the correct name, and the target OS shall issue an AVC notification to the requesting OS indicating a change to the manufacturer name.
String  location

This attribute identifies the geographical location of the EquipmentHolder.
String  acceptableEquipmentTypeList

This attribute identifies the types of equipment that can be supported by the EquipmentHolder.
String  type

This attribute identifies the type of the Holder (e.g., Rack (or Bay), Shelf, Sub-shelf, Slot, Subslot, Remote-unit or Remote-subslot).
