Type Definitions
Data Type SubnetworkConnectionSetData

This data type contains the parameters that can be passed when a Subnetwork Connection is created or modified.

Attributes
 direction

This attribute indicates the direction of the Subnetwork Connection.
 staticProtectionLevel

This attribute indicates the degree of internal resilience/protection of the Subnetwork Connection.
ProtectionEffort  protectionEffort

This attribute indicates whether the resilience as stated in the staticProtectionLevel attribute must be achieved or not.
 rerouteAllowed

This attribute specifies if the target OSs/Managed Elements are allowed and/or required to reroute this Subnetwork Connection (SNC) if there is a failure on this SNC, periodically to optimize the Routes, or for any other reason. It is a target OS/Managed Element implementation whether this is done using network routing protocols or if the target OSs/Managed Elements detect the failure and take appropriate action to attempt to fix the SNC.
 networkRouted

This attribute specifies if the network is allowed/required to route this Subnetwork Connection.
 networkReroute

This attribute indicates if the reroute (if allowed) should be computed by the network (YES), by the target OS (NO), or if it does not matter (NOT_SET).
 type

This attribute indicates the external traffic flow of the Subnetwork Connection through the Multi Layer Subnetwork.
 layerRate

This attribute indicates the layer at which the Subnetwork Connection (SNC) is to be made.
Note: In case of creation of a bundled SNC the value LR_Not_Applicable has to be used. The target OS determines the layer rate(s) by looking at the Connection Termination Points contained in the a End and z End Group Termination Points.
Boolean  isFullRoute

This attribute indicates whether the provided inclusion constraints specifies the complete route of the Subnetwork Connection (as opposed to only a partial constraint). When no inclusions constraints are specified false must be used.
Boolean  isReportingAlarms

This attribute indicates whether alarm reporting for the Subnetwork Connection is administratively activated or de-activated.
True = alarm reporting is activated; false = alarm reporting is de-activated.
Boolean  mustRemoveGtps

This attribute defines that the target OS has to delete all the interior Group Termination Points (GTPs) supporting the bundled Subnetwork Connection (SNC) when the SNC is deleted. The attribute is only used when creating bundled SNCs.
It can take on two possible values:
- true (the target OS must make a best-effort attempt to remove the interior GTPs that once belonged to the SNC) and
- false (the requesting OS doesn't care if the target OS removes the interior GTPs or not).
In the case that mustRemoveGtps is set to "true" and the target OS cannot remove all the interior GTPs, the target OS should return a list of the GTPs that could not be removed in the errorReason field of the deactivateAndDeleteSnc and deleteSnc operations. The errorReason will be "not all interior GTPs could be deleted" - this will be followed by the names of the GTPs that could not be deleted.
Boolean  isRevertive

This attribute defines if the Subnetwork Connection to be created can be switched back to its original (intended) Route.
Integer  priority

This attribute indicates the requested priority of the Subnetwork Connection to be created.
Allowed values are = "0", "1", "2", ... where zero (0) is the highest priority.
 aEndTpRoleList

This attribute indicates the end point role of the aEndTPs of the Subnetwork Connection (SNC).
At a connection matrix (CM) end point the span of the SNC starts at a G.805 Connection Point (CP) with a fixed or flexible connection through the Managed Element at the SNC layer, i.e. the connectable layer of the end point. At a link connection (LC) end point connectivity is adapted from a server layer and the span of the SNC starts with a G.805 Termination Connection Point (TCP).
 zEndTpRoleList

This attribute indicates the end point role of the zEnd TPs of the Subnetwork Connection (SNC).
At a connection matrix (CM) end point the span of the SNC starts at a G.805 Connection Point (CP) with a fixed or flexible connection through the Managed Element at the SNC layer, i.e. the connectable layer of the end point. At a link connection (LC) end point connectivity is adapted from a server layer and the span of the SNC starts with a G.805 Termination Connection Point (TCP).
 aEndTpRefList

This attribute indicates the names of the aEnd Termination Point(s). This is also used to indicate the Source TP when adding a leg to an existing broadcast system. If the TP is an FTP the requesting OS is allowed to specify a generic end point within a Managed Element, the target OS will choose the appropriate TP instance.

Traceability: R_TMF518_RP_II_0004.
 zEndTpRefList

This attribute indicates the names of the zEnd Termination Point(s). This is also used to indicate the Sink TP when adding a leg to an existing broadcast system. If the TP is an FTP the requesting OS is allowed to specify a generic end point within a Managed Element, the target OS will choose the appropriate TP instance.

Traceability: R_TMF518_RP_II_0004.
 ccInclusionList

This attribute specifies a list of Cross-Connects that must be used by the Subnetwork Connection.
 exclusionResourceRefList

This attribute specifies a list of Termination Points, Subnetwork Connections, Managed Elements, Topological Links and Group Termination Points that must not be used by the Subnetwork Connection.
The list must be empty if no exclusion constraints are required.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if inclusionResourceRefList or ccInclusionList is not empty.
 inclusionResourceRefList

This attribute specifies a list of Termination Points, Managed Elements, Topological Links, Group Termination Points and server Subnetwork Connections that must be used by the Subnetwork Connection.
The list must be empty if no inclusion constraints are required.
If the target OS cannot fully satisfy the routing constraints provided, then the request will be rejected.
Specifying both inclusion and exclusion constraints is not supported, therefore this list must be empty if exclusionResourceRefList is not empty.
 asapRef

This attribute indicates the assignment of an Alarm Severity Assignement Profile (ASAP) to the Subnetwork Connection.
