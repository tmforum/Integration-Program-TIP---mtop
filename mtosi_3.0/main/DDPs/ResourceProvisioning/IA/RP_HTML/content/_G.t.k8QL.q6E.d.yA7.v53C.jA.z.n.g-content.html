Type Definitions
Data Type SubnetworkConnectionModifyData

This data type contains the modify parameters that can be passed when a Subnetwork Connection has to be modified.

Attributes
ModifyType  modifyType

This attribute specifies the class of modification; i.e. rerouting or add/remove protection.
Boolean  mustRetainOldSnc

This attribute can request to keep the old Subnetwork Connection in pending state.
Boolean  isModifyServersAllowed

This attribute identifies whether the target OS is allowed to modify the server layers to fulfil the protection constraints.
 addedOrNewRoute

Depending on the modifyType, this attribute describes the Route of a new protection leg or the whole Subnetwork Connection. When it describes a segment to be added, either the SNCP Cross-Connects or the switch TPs that will be changed in the segment may be specified by the requesting OS. The target OS then chooses the missing segments. Alternatively, the requesting OS may specify the full Route.
 removedRoute

This attribute contains the protection legs that shall be removed from the original Subnetwork Connection. Either the last Cross-Connects (that contain the SNCP) are specified by the requesting OS or the full Route may be specified.This parameter can be used in conjonction with addedOrNewRoute only to reroute a segment.
