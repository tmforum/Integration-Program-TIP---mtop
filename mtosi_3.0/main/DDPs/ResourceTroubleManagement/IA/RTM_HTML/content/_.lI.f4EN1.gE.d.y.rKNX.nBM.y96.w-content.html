Type Definitions
Data Type ActiveAlarmFilter

This data type contains parameters for filtering Alarms. Default interpretation (filter criteria is missing or is an empty list) indicates that it is not relevant to the query; i.e., all Alarms are applicable.
Examples:
#1: All filter criteria missing -> Retrieves all Alarms associated with all Managed Elements and this OS.
#2: source = INTERNAL, all other criteria missing -> Retrieves all Alarms associated with this OS only.
#3: source = EXTERNAL, all other criteria missing -> Retrieves all Alarms associated with the Managed Elements only.

Attributes
Source  source

This attribute filters the Alarms based on their source (with respect to the OS receiving this request). Possible values are INTERNAL, EXTERNAL, NOT_APPLICABLE.
The value NOT_APPLICABLE means there is no filtering on the Alarm source; i.e., internal and external alarms are in the scope.
 scope

This attribute filters the Alarms based on a list of Managed Elements. The listed Managed Elements are in the scope.
If no Managed Element is provided, Alarms from all Managed Elements are in scope.
Note: In case of remote units, the Managed Elements could also be addressed by providing the identifiers of Equipment Holders.
 perceivedSeverityList

This attribute filters the alarms based on a list of severities (see ITU-T X.733 definitions). The listed severities are in the scope.
If no severity is provided, alarms with any severity are in scope.
String  probableCauseList

This attribute filters the Alarms based on a list of Probable Causes (for details refer to the supporting document SD1-33_ProbableCauses). The listed Probable Causes are in the scope.
If no Probable Cause is provided, Alarms with any Probable Cause are in scope.
AcknowledgeIndication  acknowledgeIndication

This attribute filters the Alarms based on its acknowledge indication. Possible values are ACKNOWLEDGED, UNACKNOWLEDGED, and NOT_APPLICABLE.
The value NOT_APPLICABLE indicates that Alarms with any acknowledge indication are in scope.
