Interface ResourceInventoryRetrievalService

Operations
 getInventory

This operation returns a specified portion of the inventory data from the OS to which the request is directed (i.e., the target OS). The target OS returns all objects (in batches) satisfying the scope and filter constraints of the requesting OS that have a modification date equal to or later than the Diff Date and Time.
The operation can only be directed to a top-level OS and not to a subordinate OS.

Traceability: R_TMF518_MRI_II_0002, R_TMF518_MRI_II_0003, R_TMF518_MRI_II_0004, UC_TMF518_MRI_0011.
 getRemoteUnitProfiles

This operation allows the requesting OS to retrieve the non-layer oriented profiles assigned to a remote unit (e.g., DSL NT, GPON ONT).

 getInventory 
Public  getInventory(, InventoryFilter, ,  )

This operation returns a specified portion of the inventory data from the OS to which the request is directed (i.e., the target OS). The target OS returns all objects (in batches) satisfying the scope and filter constraints of the requesting OS that have a modification date equal to or later than the Diff Date and Time.
The operation can only be directed to a top-level OS and not to a subordinate OS.

Traceability: R_TMF518_MRI_II_0002, R_TMF518_MRI_II_0003, R_TMF518_MRI_II_0004, UC_TMF518_MRI_0011.

Parameters:
diffDateAndTime [1]This parameter allows to constraint the inventory retrieval by a modification date and time. I.e., all objects with a modification date equal to or greater than this value are in scope of this operation.
InventoryFilter filter [1]This parameter defines the subset of the total inventory to be returned.

Traceability: R_TMF518_MRI_II_0004.
inventoryData [1]This parameter returns the requested inventory data.
The inventory data may vary based on the MEP. Refer to the attached supporting document SD2-5_Communication_Styles.
whereAndHowToReply [1]This parameter contains the necessary information (including where and how) the target OS needs to send the response. The information provided in this parameter depends on the scenario (i.e., type of communication style) used for retrieving the network inventory. Refer to the attached SD2-5_Communication_Styles supporting document. The individual information to be provided by the requesting OS is described in the following Use Cases:
- UC_TMF518_MRI_0011: Bulk Inventory Retrieval - Scenario 1: Coarse grained approach, batched response, message communication style
- UC_TMF518_MRI_0012: Bulk Inventory Retrieval - Scenario 2: Coarse grained approach, RPC communication style
- UC_TMF518_MRI_0013: Bulk Inventory Retrieval - Scenario 3: Coarse grained approach, file transfer.
