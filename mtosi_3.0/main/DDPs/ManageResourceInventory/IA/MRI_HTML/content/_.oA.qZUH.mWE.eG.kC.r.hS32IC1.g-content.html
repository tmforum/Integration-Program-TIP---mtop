Interface MultiLayerSubNetworkPointPoolLinkRetrievalService

 getAllMlsnppLinksWithTp

This operation is used to query all the MultiLayer SNPP Link(s) that are known to the target OS using the specified Termination Point (PTP or CTP).
Both internal and edge links are returned.
It can be requested to return also the SNPs terminating the MLSNPP Links to be returned.

Traceability: R_TMF518_MRI_II_0189, UC_TMF518_MRI_0021.

 getAllMlsnppLinks

This operation returns all MultiLayerSNPPLinks known to the target OS.

If a MultiLayerRoutingArea object name is provided as an input parameter, the operation returns only the MultiLayerSNPPLinks internal and external to the specified MultiLayerRoutingArea (only MultiLayerSNPPLinks directly within the MultiLayerRoutingArea or directly at the edge of the MultiLayerRoutingArea are returned).

If the MultiLayerRoutingArea represents a Routing Node, then only external MultiLayerSNPPLinks are returned.
The MultiLayerSNPPLinks returned may include the information on the SNPs, dependent upon the input parameter choice.

Considering the external MultiLayerSNPPLinks:
- In the case of the top-level MultiLayerRoutingArea then the operation shall return all the MultiLayerSNPPLinks to external devices (i.e. UNI or dumb-SNPPLinks - note that dumb-SNPPLinks may coincide with top-level Topological Links).
- In the case of a MultiLayerRoutingArea of level N, N being any level subordinate of the top level (including the Routing Node) then it shall return all the MultiLayerSNPPLinks to another MultiLayerRoutingArea of the same level N.

Considering the internal MultiLayerSNPPLinks:
- Only I-NNI links are returned.
Note that E-NNI and I-NNI definitions are related to the hierarchy level considered, following the partitioning concept. For example. given a MultiLayerRoutingArea of level N, its edge links have the E-NNI role with respect to level N. Same links have I-NNI role with respect to level N-1 (superior), i.e. are internal links of superior MultiLayerRoutingArea.
The reply may be an empty or a partial list depending upon the visibility that the target OS has of the control plane:
- Local & Complete - the target OS completely manages the MultiLayerRoutingArea and the supporting transport plane resources (ME, TP, etc.) such that it can provide a complete view of the MultiLayerSNPPLinks and all associated transport plane data.
- Local CP Only - the target OS manages the MultiLayerRoutingArea but has no view of the supporting transport plane resources such that it can provide a complete view of the MultiLayerSNPPLinks but cannot provide any associated transport plane data.
- Remote CP Transparent - the target OS manages the MultiLayerRoutingArea at this level but may not have a view of the details of the subordinate MLRA levels and has no view of the supporting transport plane resources, such that it can provide a complete view of the MultiLayerSNPPLinks but not necessarily all of the SNPP aliases and can not provide any associated transport plane data.
- Remote CP Limited - the target OS partially manages the MultiLayerRoutingArea at this level and may not have a view of the details of the subordinate MultiLayerRoutingArea levels and has no view of the supporting transport plane resources, such that it can provide a view of some of the MultiLayerSNPPLinks and not necessarily all of the subordinate SNPPs for those MultiLayerSNPPLinks and can not provide any associated transport plane data.
- Remote CP Opaque - the target OS is unable to provide any information related to MultiLayerSNPPLinks that are internal to the specified MultiLayerRoutingArea.

Traceability: R_TMF518_MRI_II_0187, UC_TMF518_MRI_0021.
 getAllInternalMlsnppLinks

This operation has exactly the same behaviour as the getAllMultiLayerSubNetworkPointPoolLinks operation, but instead of returning all the Multi Layer SNPP Link objects this operation returns only those that are internal to the addressed Multi Layer Routing Area.
If the addressed Multi Layer Routing Area is a Routing Node then the list will be empty.

Traceability: R_TMF518_MRI_II_0186, UC_TMF518_MRI_0022.

 getAvailableCapacity

This operation retrieves the capacity of the MulitLayer SNPP Link in terms of a count of the connections available on a per layer basis. If a layer rate is not specified then the target OS returns the available capacity for each layer rate that is supported.
