Operations
 getAllAssignableConnectionlessPortTerminationPoints

This operation retrieves the details of all Connectionless Port Termination Points (CPTPs) which are potentially able to be assigned to the provided Matrix Flow Domain (MFD).
Potentially means: The CPTPs are on the same Equipment or same Rack with backplane connectivity as the MFD. It is irrelevant whether the CPTPs are already assigned to an MFD or not.

Traceability: R_TMF518_MRI_II_0116.
 getAllAssignedConnectionlessPortTerminationPoints

This operation retrieves the details of all Connectionless Port Termination Points assigned to the provided Matrix Flow Domain.

Traceability: R_TMF518_MRI_II_0111.
 getAllConnectionlessPortTerminationPoints

This operation retrieves the details of all Connectionless Port Termination Points (CPTPs) associated to the provided Flow Domain. It is possible to filter the request to retrieve only fdEdge (UNI and E-NNI sides), only fdInternal or all CPTPs.

Traceability: R_TMF518_MRI_II_0118.
 getAllEdgePoints

This operation retrieves a filtered list of all Termination Points (TPs) that are at the edge of the provided Multi Layer Subnetwork (MLSN) which
- contain one or more of the provided TP Layer Rates and
- are capable of containing CTPs that can be connected at one or more of the also provided connection Layer Rates.
The operation considers the capability/flexibility of the PTPs, not their current states.

Traceability: R_TMF518_MRI_II_0064, R_TMF518_MRI_II_0066.
 getAllFloatingTerminationPoints

This operation returns a filtered list of all Floating Termination Points (FTPs) within a Managed Element (ME) which
- contain one or more of the provided TP Layer Rates and
- are capable of containing CTPs that can be connected at one or more of the provided connection Layer Rates.
The operation considers the capability/flexibility of the FTPs, not their current states.

Traceability: R_TMF518_MRI_II_0076, R_TMF518_MRI_II_0078.
 getAllGroupTerminationPoints

This operation returns the details of all Group Termination Points (GTPs) within the provided ManagedElement (ME) that contain one or more TPs at the provided Layer Rates.
If there are no GTPs that match the layer constraints, an empty list is returned.

Traceability: R_TMF518_MRI_II_0102.
 getAllPhysicalTerminationPoints

This operation retrieves a filtered list of all Physical Termination Points (PTPs) and Floating Termination Points (FTPs) within a provided Managed Element (ME) which
- contain one or more of the provided TP Layer Rates and
- are capable of containing CTPs that can be connected at one or more of the provided connection Layer Rates.
The operation considers the capability/flexibility of the PTPs/FTPs, not their current states.

Traceability: R_TMF518_MRI_II_0059, R_TMF518_MRI_II_0061.
 getAllPhysicalTerminationPointsWithoutFtps

This operation retrieves a filtered list of all Physical Termination Points (PTPs) (no Floating Termination Point (FTP) subclasses) within a provided Managed Element (ME) which
- contain one or more of the provided TP Layer Rates and
- are capable of containing CTPs that can be connected at one or more of the provided connection Layer Rates.
This operation considers the capability/flexibility of the PTPs, not their current states.

Traceability: R_TMF518_MRI_II_0055, R_TMF518_MRI_II_0057.
 getAllSupportedPhysicalTerminationPoints

This operation retrieves all Physical Termination Points (PTPs) and Floating Termination Points (FTPs) that are directly supported by a provided Equipment.
The PTPs that are returned are those that share their physical layer with the primary Equipment (i.e. that represent a Port on the Equipment or are connected by a fibre, wire, etc.).
The FTPs that are returned are those which are implemented by the physical circuitry supported by the Equipment and will include FTPs that are named from the provided Equipment (the FTPs will depend upon the presence of that Equipment for their most basic operation).
When there is Equipment Protection, the operation reports PTPs/FTPs for the primary Equipment only. That is, when called on a protecting Equipment (as opposed to the primary Equipment), the operation returns an empty list, irrespective of the current switch status.

Traceability: R_TMF518_MRI_II_0081.
 getAllTerminationPointPools

This operation retrieves all Termination Point Pools (TP Pools) contained by a provided Multi Layer Subnetwork (MLSN).

Traceability: R_TMF518_MRI_II_0052.
 getAssociatedTerminationPoints

This operation retrieves a list of PSR (UPSR or SNCP) associated Termination Points (TPs) for the TP whose name is provided.
The operation retrieves the associated CTP of a given CTP or the associated PTP of a given PTP. The timeslot of the TPs will be the same in case of an OPEN_PSR Multi Layer Subnetwork, but the names of the two TPs will be different.
The operation is symmetric on the associated TPs. Given a working TP, the associated TP will be the protecting TP.
When the operation is invoked with a TP that is not an edge point, the returned TPs will be on the same Managed Element.

Traceability: R_TMF518_MRI_II_0080.
 getAssociatedTerminationPointsWrtTmd

This operation retrieves the list of Termination Points (TPs) that are associated with a provided Transmission Descriptor (TMD).

Traceability: R_TMF518_MRI_II_0048, R_TMF518_MRI_II_0050.
 getContainedCurrentConnectionTerminationPoints

This operation retrieves the "current" Connection Termination Points (CTPs) which
- are contained within the provided Termination Point (PTP or CTP) and
- contain one or more of the provided Layer Rates.
A "current" CTP is defined as a CTP that is either cross-connectible or cross-connected, in the current mapping configuration.
Examples for contained Current TPs in different states of usage are contained in the associated supporting document SD1-6_ContainedTPs.

Traceability: R_TMF518_MRI_II_0070.
 getContainedInUseConnectionTerminationPoints

This operation retrieves the "in use" Connection Termination Points (CTPs) which
- are contained within the provided Termination Point (PTP or CTP) and
- contain one or more of the provided Layer Rates.
An "in use" CTP is defined as a CTP that is used by a Subnetwork Connection (SNC) in any state (including pending), either as a CM end point or as an intermediate connection point, or a CTP that is terminated and mapped.
The operation can be used when there are a large number of potential CTPs (e.g., in ATM).
Examples for contained InUse TPs in different states of usage are contained in the associated supporting document SD1-6_ContainedTPs.

Traceability: R_TMF518_MRI_II_0072.
 getContainedPotentialConnectionTerminationPoints

This operation retrieves the "potential" Connection Termination Points (CTPs) which
- are contained within the provided Termination Point (PTP or CTP) and
- contain one or more of the provided Layer Rates.
Potential CTPs are all CTPs supported by all possible mapping configurations. If the layerRateList is empty then contained CTPs at all flexible and/or static LayerRates are returned.
Examples for contained Potential TPs in different states of usage are contained in the associated supporting document SD1-6_ContainedTPs.
Note that in any case, this operation returns the complete object information for all contained CTPs, not just those CTPs that are directly contained by the provided TP.

Traceability: R_TMF518_MRI_II_0068.

 getTerminationPointGroupingRelationships

This operation queries the grouping associations between Termination Points (TPs), Termination Point Pools (TP Pools) and Group Termination Points (GTPs).
It allows to retrieve:
- the names of all TP Pools and GTPs associated to a provided TP; (R_TMF518_MRI_II_0049)
- the name of the TP Pool containing a provided GTP; (R_TMF518_MRI_II_0106)
- the names of all TPs and GTPs contained in a provided TP Pool; (R_TMF518_MRI_II_0054).
Note: A TP can be a member of at most one TP Pool. A TP is associated to a TP Pool (TP --> TP Pool) by admission as a member (TP is a member of TP Pool). The association of a TP Pool to a TP (TP Pool --> TP) can be, but need not be, the reversal of the membership association of a TP to a TP Pool (TP Pool contains TP). Therefore several TP Pools can be associated to a TP.
Descriptions of the associations can be obtained from the respective TP Pool attributes for the description of use.
An example for an association, that is not the containment of the TP itself in a TP Pool, is the partitioning of the set of potential client CTPs of an ATM NI CTP (real user network interface) into a set of ATM VP TP pools (virtual user network interfaces). See attached supporting document SD1-41_TPPoolRelationship.

Traceability: R_TMF518_MRI_II_0049, R_TMF518_MRI_II_0054, R_TMF518_MRI_II_0106.

 getAssociatedTerminationPoints 
Public  getAssociatedTerminationPoints(,  )

This operation retrieves a list of PSR (UPSR or SNCP) associated Termination Points (TPs) for the TP whose name is provided.
The operation retrieves the associated CTP of a given CTP or the associated PTP of a given PTP. The timeslot of the TPs will be the same in case of an OPEN_PSR Multi Layer Subnetwork, but the names of the two TPs will be different.
The operation is symmetric on the associated TPs. Given a working TP, the associated TP will be the protecting TP.
When the operation is invoked with a TP that is not an edge point, the returned TPs will be on the same Managed Element.

 getTerminationPointGroupingRelationships 
Public  getTerminationPointGroupingRelationships(,  )

This operation queries the grouping associations between Termination Points (TPs), Termination Point Pools (TP Pools) and Group Termination Points (GTPs).
It allows to retrieve:
- the names of all TP Pools and GTPs associated to a provided TP; (R_TMF518_MRI_II_0049)
- the name of the TP Pool containing a provided GTP; (R_TMF518_MRI_II_0106)
- the names of all TPs and GTPs contained in a provided TP Pool; (R_TMF518_MRI_II_0054).
Note: A TP can be a member of at most one TP Pool. A TP is associated to a TP Pool (TP --> TP Pool) by admission as a member (TP is a member of TP Pool). The association of a TP Pool to a TP (TP Pool --> TP) can be, but need not be, the reversal of the membership association of a TP to a TP Pool (TP Pool contains TP). Therefore several TP Pools can be associated to a TP.
Descriptions of the associations can be obtained from the respective TP Pool attributes for the description of use.
An example for an association, that is not the containment of the TP itself in a TP Pool, is the partitioning of the set of potential client CTPs of an ATM NI CTP (real user network interface) into a set of ATM VP TP pools (virtual user network interfaces). See attached supporting document SD1-41_TPPoolRelationship.

Traceability: R_TMF518_MRI_II_0049, R_TMF518_MRI_II_0054, R_TMF518_MRI_II_0106.
