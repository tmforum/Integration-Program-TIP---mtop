Type Definitions
Data Type InventoryFilter

This data type contains the Inventory Filter structure used for the getInventory() operation.

Traceability: R_TMF518_MRI_II_0004.

Attributes
String  baseInstanceList

This attribute identifies a list of base object instances for the subtrees to be retrieved. It is a list of serialized MTOSI names. If the list is left empty then all the nodes at the top of the MTOSI instance tree are selected (i.e., all MDs, TMDs, inter-MD-TLs, and OSs are selected). Specification of the baseInstanceList does not select any objects in and of itself. In order to select specific objects, one needs to specify the IncludedObjectTypes.
The ";" is used as a separator, and "=" is used to associate a value with a type of object. For example, MD=md1;ME=ME123;EH=ACME7 identifies an equipment holder within managed element ME123 within management domain md1. The relative name of the equipment holder is ACME7.
The following abbreviations are used for the object type names:
MD - management domain
ME - managed element
EH - equipment holder
EQ - equipment
PTP - physical termination point
FTP - floating termination point
CTP - connection termination point
TPPool - termination point pool
EPGP - equipment protection group
PGP - protection group
SNC - subnetwork connection
MLSN - multi layer subnetwork
TL - topological link
OS - operations system
TMD - transmission descriptor.
IncludedObjectType  includedObjectTypeList

This attribute identifies the type of objects contained under the selected base object instances that are to be included in the returned inventory file. The requesting OS always have to provide at least one object. The base object itself may also be included.

For each included object type, one needs to indicate whether
a. just the name,
b. all the associated attributes (including the name), or
c. the associated attributes and relationship pointers
are to be returned for each instance.

The includedObjectTypes need to be picked such that each selected base class has a single (connected) subtree below it.

The following abbreviations are used for the object type names:
MD - management domain
ME - managed element
EH - equipment holder
EQ - equipment
PTP - physical termination point
FTP - floating termination point
CTP - connection termination point
TPPool - termination point pool
EPGP - equipment protection group
PGP - protection group
SNC - subnetwork connection
RT - route
CC - cross connection
MLSN - multi layer subnetwork
TL - topological link
OS - operations system
TMD - transmission descriptor.
