Type Definitions
Data Type DiversityViolations

This data type defines the diversity violation related parameters.

Attributes
DiversitySynthesis  diversitySynthesis

This attribute indicates a synthesis of the current state of diversity violations of the Call (diversity which is different from the provisioned diversity).

Possible values are:
- Not Applicable
- Unknown
- NoViolations
- Violations (in case the EMS does not distinguish between node and link violations)
- LinkViolations
- NodeViolations
- LinkAndNodeViolations.
DiversityInfo  linkDiversityViolations

This attribute contains the list of link diversity violations.

The value srgType indicates the link SRG type required for the call.
The sharedResourceList indicates the MLSNPPLink/TopologicalLink Name(s) where the violation(s) occurred and also the Name of the connection(s)/SNC(s) affected by each violation.

Note that the connection(s) affected by the violation(s) can not be currently routed in the case of a Mandatory diversity effort requirement.
Depending upon the capability of the target OS this parameter may be empty.
DiversityInfo  linkPartialDiversityList

This attribute contains the link diversity that was partially achieved in the cases where the diversity requirements were not met. It provides further information on the current state of routing.

The parameter provides three possible levels of detail:
- SRG types where the violation is, plus (opt.) the related
- list of links involved, plus (opt.) the related
- list of involved Connections.

The value srgType indicates:
- For Mandatory: The SRG type that could have been achieved on the links(s) where the diversity violation occurred if the request had been for Best Effort. The sharedResourceList indicates the MLSNPPLink/TopologicalLink(s) where the "reduced" diversity would have been achieved and also the name of each top level Connection affected.
- For Best Effort: The SRG type that has been achieved on the links(s) where the diversity violation occurred. The sharedResourceList indicates the MLSNPPLink/TopologicalLink(s) where the "reduced" diversity has been achieved and also the Name of each top level connection affected.

Note that in the case of a Mandatory Diversity Effort, the connection(s) affected by the violation(s) can not be currently routed and they are in "searching" state.
Depending upon the capability of the target OS this parameter may be empty.
For example, consider the case where a call is requested with best effort link diversity of "bridge" sRGType, but only routes that all share a single bridge are available. In this case a violation will be indicated identifying the links that traverse the bridge, the connections that used these links and the degree of diversity achieved (e.g. "duct").
