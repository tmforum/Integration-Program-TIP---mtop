Type Definitions
Data Type MatrixFlowDomainFragment

This data type represents the portion of a Flow Domain Fragment within a Matrix Flow Domain (MFD).
Traceability: R_TMF518_NRF_I_0059 and R_TMF518_NRF_I_0060.

Attributes
ConnectionDirection  direction

This attribute represents the directionality of the MFDFr.
Boolean  isFlexible

This attribute indicates whether the MFDFr is fixed (= True) or is flexible (= False).
Boolean  isActive

This attribute indicates if the MFDFr has been successfully activated in the Network Element (NE). True if the MFDFr is active in the ME; false otherwise. An "active" MFDFr means that no further intervention on the MFDFr itself is required from the requesting OS to activate it. The fact that an MFDFr is "active" does not necessarily imply that there is traffic flow on the network.
FlowDomainFragmentType  mfdfrType

This attribute indicates the type of the MFDFr.
 transmissionParameterList

This attribute contains the technology-specific parameters associated with the different connectionless layers that are supported by the MFDFr.
Refer to to chapter “Connectionless Technology Parameters” of the attached supporting document SD1-16_LayeredParameters for details of the currently defined Transmission Parameters.
 additionalInformation

This is a vendor specific attribute that contains additional specific information about the MFDFr. The list can be empty.
«mTOPClass» ConnectionTerminatioPoint  aEndTpRefList

This relationship contains the the A end Flow Point names.
«mTOPClass» ConnectionTerminatioPoint  zEndTpRefList

This relationship contains the the Z end Flow Point names.
