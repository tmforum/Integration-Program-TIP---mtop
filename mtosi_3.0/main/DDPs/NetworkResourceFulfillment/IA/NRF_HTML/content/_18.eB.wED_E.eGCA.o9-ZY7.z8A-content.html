Class Connection

The Connection object represents the relationship between two Subnetwork Points (SNP) or Termination Points.

A Connection may encompass a sequence of Connections (which are referred to as Connection Segments, but not represented in this model) and thus create a recursive relationship between Connections (i.e., Connections supporting Connections). The Connection(s) at the highest level of recursion are referred to as top-level Connection(s).

Note that a Call may have one or more parallel top-level Connection(s).

Traceability: R_TMF518_NRF_I_0081, R_TMF518_NRF_I_0082, R_TMF518_NRF_I_0083.

ConnectionDirection  direction

This attribute identifies the directionality of the Connection.
RerouteAllowed  rerouteAllowed

This attribute specifies if the target OS/MEs/Control Plane is allowed and/or required to reroute the Connection if there is a failure on the Connection, periodically to optimize the routes, or for any other reason. It is an target OS/ME/Control Plane implementation whether this is done using network routing protocols or if the target OS/ME/Control Plane detects the failure and takes appropriate action to attempt to fix the Connection.

The NO value means that the target OS/Control Plane is not allowed to reroute the Connection.
The YES value means that the target OS/Control Plane is allowed to reroute the Connection and is required to attempt to reroute it upon failure.
The NOT_APPLICABLE value is used when the requesting OS does not want to specify the exact target OS/Control Plane behaviour. In this case it is left up to the target OS/Control Plane to decide whether rerouting will be provided. It is also used if the target OS/Control Plane is allowed to reroute the Connection but not required to attempt to reroute it upon failure.
StaticProtectionLevel  staticProtectionLevel

This attribute indicates the degree of internal resilience/protection of the Connection.
 maximumCost

This attribute defines a number that indicates the maximum cost allowed for a Connection. The cost of a Connection is calculated based on the cost of the supporting MLSNPP links. This link cost represents a vector of one or more metrics, each of which indicates the relative desirability of a particular link over another during path selection.

ConnectionType  connectionSetupType

This attribute identifies the setup type of the Connection. It indicates whether the Connection has been established by UNI signalling (SC) or requesting OS operation (SPC, PC).
ConnectionState  connectionState

This attribute represents the state of the Connection by indicating that either all ("Complete") or not all ("Searching") of the resources necessary to support the Connection have been successfully allocated yet.

Traceability: R_TMF518_NRF_I_0084.
Boolean  revertive

This attribute indicates whether the Connection shall always attempt to return to its home (intended) Route.
Integer  priority

This attribute defines the priority of the Connection.
The priority setting allows one to prevent the activation of routes because a XC or CTP conflicts with equal or higher prior Connection.
A Connection in the Control Plane can pre-empt resources of another Connection of lower priority for restoration purposes.
Allowed values are = "0" , "1" , "2" , ... where zero (0) is the highest priority.
Boolean  isReportingAlarms

This attribute identifies whether alarm reporting for this Connection is administratively activated or de-activated.
True = alarm reporting is activated; False = alarm reporting is de-activated.
 asapRef

This attribute indicates the assignment of an Alarm Severity Assignment Profile (ASAP) to the Connection.
Writeable by the requesting OS only via assignASAP/deassignASAP operations.
String  routeGroupLabel

This attribute represents the Route Group the Connection is associated to. When a target OS creates a Call with diverse Route Groups, it assigns a Route Group label (which may be a number) to each Connection. Connections in the same Route Group will be assigned the same value for routeGroupLabel. Connections in different Route Groups will be assigned different values for routeGroupLabel.
