Type Definitions
Data Type TerminationPointData

This data type identifies a Termination Point with associated parameters and their values.
In a Control Plane Connection this data type provides the information of the SNP.

Traceability: R_TMF518_RP_II_0008.

Attributes
 tpRef

This attribute contains the Termination Point name to which the transmission parameters applies. The Termination Point name must be explicit (a generic endpoint specification may not be used in this case).
TerminationMode  tpMappingMode

This attribute indicates if the Termination Point is to be configured such that it is capable of supporting lower rate connections or is capable of supporting connections at the CTP's layer rate.
In the context of a control plane connection, the value of this attribute shall be ignored.
 transmissionParameterList

This attribute contains the layered Transmission Parameters associated with the Termination Point.
Refer to the attached supporting document SD1-16_LayeredParameters for details of the currently defined Transmission Parameters.
When used as input to the target OS, this is a "delta" list that needs to be applied to the specified TP. Only a subset of the parameters may be specified in the list, and only those should be applied, typically in the NE.

Note that this could be used to convey the pointer to the alarm severity assignment profile during SNC creation/activation.
If the list is empty then this means do nothing. To remove a parameter from the list, "-" should be specified in the value part of the structure.
When a BLSR connection is created, the SPRING_NodeId parameter should be provided for the aEnd and zEnd CTPs/FTPs.
 ingressTmdRef

This attribute indicates the name of the ingress Traffic Descriptor or Transmission Descriptor that supports this CTP.
May be empty if the CTP is involved in a uni-directional connection.
A connection termination point may have an optional reference to an ingress (incoming) Transmission Descriptor. The Descriptor name will be empty if there is no associated Descriptor, e.g., when CTP is involved in a uni-directional connection.
It is expected that Traffic Descriptors and Transmission Descriptors are not mixed on a single TP. Therefore if the Ingress Descriptor name is that of a Traffic Descriptor then the Egress Descriptor name should either be that of a Traffic Descriptor or a null value.
Likewise the other case for Transmission Descriptor.
 egressTmdRef

This attribute indicates the name of the egress Traffic Descriptor or Transmission Descriptor that supports this CTP.
May be empty if the CTP is involved in a uni-directional connection.
A connection termination point may have an optional reference to an egress (outgoing) Transmission Descriptor. The Transmission Descriptor name will be empty if there is no associated Descriptor, e.g., when CTP is involved in a uni-directional connection.
