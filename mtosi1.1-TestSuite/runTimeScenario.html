Run Time Scenario

This section describes the different scenarios which are used as pre-conditions for each test case. To make those scenarios more understandable, the test lab as described in [5] is used as illustrative example.

  • Scenario 1
    • Two actors
      • The EMS-OS whose unique name on the CCV is is /MTOSI/EMS01
      • The InventoryOS whose unique name on the CCV is /MTOSI/InventoryOS
    • The InventoryOS manages a network with is populated with resources like MEs, EH, Eq, PTPs, CTPs, TLs and MLSNs.
      Not all the logical resources have been created in the EMS-OS. Following [5], only the UPSR Bi-directional SNCs have been created. The UPSR Uni-directional SNCs and the Matched Node UPSR (BI) SNC have not been created yet.
  • Scenario 2
    • Same as scenario 1 but in addition the EMS-OS has been populated with new logical resources.
      Following [5], the UPSR Uni-directional SNCs and the Matched Node UPSR (BI) SNC are now present in the EMS-OS.

  • Scenario 4
    • Four actors:
      • the EMS-OS whose unique name on the CCV is /MTOSI/EMS01
      • the InventoryOS whose unique name on the CCV is /MTOSI/InventoryOS
      • the Fault-OS whose unique name on the CCV is /MTOSI/FaultOS
      • the Broker-OS whose unique name on the CCV is /MTOSI/BrokerOS
    • the EMS-OS can be in any of the three states specified in scenario 1, 2 or 3.
    • the Inventory-OS has subscribed to the notification service to receive the Inventory Topic notifications generated by the EMS-OS, with a selector specifying interest only in the ObjectDeletion notification
    • the Inventory-OS has also subscribed to the notification service to receive the Fault Topic notifications generated by the EMS-OS
    • the Fault-OS has subscribed to the notification service to receive the Fault Topic notifications generated by the EMS-OS.
  • Scenario 5
    • Same as scenario 4, but the Inventory-OS has changed its selector to the Inventory Topic notifications generated by the EMS-OS: the selector specifies that the Inventory-OS is willing to receive three kinds of Inventory Topic notifications: ObjectCreation, ObjectDeletion and Attribute Value Change.
