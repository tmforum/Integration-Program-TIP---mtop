Test Identification Test Number: 4
Test Category: Configuration Service
MTOSI Interface: InventoryRetrieval
Test Name: Multiple Batch Response - RPC Style
517 RequirementRequirement II. 9
Operations getInventory
Preconditions
Scenario 1
Test case Request:
Header Parameters:
CommunicationPattern = MultipleResponse
CommunicationStyle = RPC
RequestedBatchSize = 2
Body Parameters:
Filter.baseInstanceList = { (MD=md1) }
Filter.includedObjectTypes = { (ME, FULL) , (EH, FULL), (PTP, FULL) }
Response:
Retrieves the attributes and relationship pointers of all Managed Element, Equipment Holder and Physical Termination Point within Management Domain md1. The requested information is sent using the RPC communication style in two batches (an iterator pattern is used).
XML Examples:
request1     response1     request2     response2    

Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
