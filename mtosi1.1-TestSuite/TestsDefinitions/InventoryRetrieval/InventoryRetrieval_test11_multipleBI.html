Test Identification Test Number: 11
Test Category: Configuration Service
MTOSI Interface: InventoryRetrieval
Test Name: Filter – Multiple Base Instances
517 Requirement Requirement II. 10 Each of the bulk inventory mechanisms listed in Requirement II 9 shall support the ability to provide a subset of the inventory known to the target OS. The subset is defined by a filter provided by the requesting OS. The following filter shall be supported:
f. Selection of all object types or a subset of the object types contained under a given base instance, including the object associated with the base instance, shall be an allowable filter condition. A single filter can have multiple base instances, i.e., multiple subtrees can be selected.
Operations getInventory
Preconditions
Scenario 2
Test case Request:
Header Parameters:
CommunicationPattern = MultipleResponse
CommunicationStyle = MSG
RequestedBatchSize = 0
Body Parameters:
Filter.baseInstanceList = { (MD=md1, ME=me1), (MD=md1, ME=me2) }
Filter.includedObjectTypes = { (ME, ATTRS), (EH, NAME) }
Response:
Retrieves the attributes of all Equipment Holder within Managed Element me1 within Management Domain md1.
Retrieves the attributes of all Equipment Holder within Managed Element me2 within Management Domain md1.
XML Examples:
request     response    

Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
