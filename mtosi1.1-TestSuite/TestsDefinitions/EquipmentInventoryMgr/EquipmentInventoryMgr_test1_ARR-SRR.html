Scenario 2
Test case Operation:     getSupportingEquipment
Request Body Parameters:
equipmentName: the name of the equipment for which to retrieve the supporting equipments.
Response:
This service allows an NMS to request all of the equipment that supports a given piece of equipment, e.g., the power pack and synchronization card that support a given piece of equipment.
XML Examples:
none        

Operation:     getSupportingEquipmentNames
Request Body Parameters:
Same as getSupportingEquipment.
Response:
This operation has exactly the same behaviour as getSupportingEquipment(), but returns the object names instead of returning the entire objects. Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     getSupportedEquipment
Request Body Parameters:
equipmentName: The name of the equipment for which to retrieve the supported equipment list
Response:
This service allows an NMS to request all of the pieces of equipment supported by a given piece of equipment, e.g., a power pack might support many circuit packs.
XML Examples:
none        

Operation:     getSupportedEquipmentNames
Request Body Parameters:
Same as getSupportedEquipment.
Response:
This operation has exactly the same behaviour as getSupportedEquipment(), but returns the object names instead of returning the entire objects. Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     getContainedEquipment
Request Body Parameters:
equipmentHolderName: The name of the equipment holder for which to retrieve the directly contained equipment and equipment holders.
Response:
This service returns the equipments and equipment holders directly contained by a specific equipment holder. This method differs from getAllEquipment in that it only looks at the next level of the containment hierarchy.
XML Examples:
request     response    

Operation:     getEquipment
Request Body Parameters:
equipmentOrHolderName: The name of the equipment or equipment holder to retrieve
Response:
This service returns the equipment or equipment holder for the given equipment or equipment holder name.
XML Examples:
request     response    

Exceptions EXCPT_NOT_IMPLEMENTED
EXCPT_INTERNAL_ERROR
EXCPT_INVALID_INPUT
EXCPT_ENTITY_NOT_FOUND
EXCPT_COMM_FAILURE
Note: Those exceptions are common to all operations.
Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
