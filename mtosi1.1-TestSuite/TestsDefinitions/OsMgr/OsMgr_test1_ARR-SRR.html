Preconditions

Test case Operation:     getAllTLsBetweenMDs
Request Body Parameters:
none
Response:
This operation returns all the TopologicalLink (TL) objects with end points in two different ManagementDomains (MDs) which are managed by the addressed top-level OS. The operation can only be directed to a top-level OS and not to a subordinate OS.
XML Examples:
none        

Operation:     getAllTLNamesBetweenMDs
Request Body Parameters:
none
Response:
This operation returns the name of each TopologicalLink (TL) with end points in two different ManagementDomains (MDs) which are managed by the addressed top-level OS. The operation can only be directed to a top-level OS and not to a subordinate OS. Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     getTopLevelTopologicalLink
Request Body Parameters:
topoLinkName: Name of the top level topological link to retrieve.
Response:
This service returns a top level topological link given its name.
XML Examples:
none        

Operation:     getOS
Request Body Parameters:
none
Response:
This operation returns the data structure associated with the addessed OS. This operation may only be directed to a top-level OS and not to a subordinate OS.
XML Examples:
none        

Operation:     getSubordinateOS
Request Body Parameters:
none
Response:
This operation returns the data structure associated with a given subordinate OS identified by the provided subordinateOSName. The operation can only be directed to a top-level OS and not to a subordinate OS.
XML Examples:
none        

Operation:     getMD
Request Body Parameters:
mdName: The name of the MD being requested.
Response:
This operation returns the data structure associated with a given Management Domain (MD) identified by the provided mdName. The operation can only be directed to a top-level OS and not to a subordinate OS.
XML Examples:
none        

Exceptions EXCPT_NOT_IMPLEMENTED
EXCPT_INTERNAL_ERROR
EXCPT_INVALID_INPUT
EXCPT_ENTITY_NOT_FOUND
EXCPT_COMM_FAILURE
Note: Those exceptions are common to all operations.The conditions when they are raised are specific to each operation.
Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
