Preconditions
Scenario 2
Test case Operation:     getAllNUTTPNames
Request Body Parameters:
ePGPName: name of the equipment protection group that the client is interested in.
Response:
This operation is used to get the current status of a Equipment Protection Group. This service is needed so that even if a filter is established, the NMS can query the status of a protection group.
XML Examples:
none        

Operation:     getAllProtectedTPNames
Request Body Parameters:
pgName: Name of the protection group
Response:
This operation gets the list of CTPs that can carry protected traffic. The result is independent of the current switch status. Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     getAllPreemptibleTPNames
Request Body Parameters:
groupName: Name of the protection group
Response:
This operation gets the list of names of CTPs that can carry preemptible extra traffic. The result is independent of the current switch status Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     getAllProtectionGroups
Request Body Parameters:
meName: the name of the managed element for which the request is made
Response:
This operation is used by the client to discover all the protection groups currently in operation for the managed element.
XML Examples:
none        

Operation:     getAllEProtectionGroups
Request Body Parameters:
meName: the name of the managed element for which the request is made
Response:
This operation is used by the client to discover all the equipment protection groups currently in operation for the managed element.
XML Examples:
none        

Exceptions EXCPT_NOT_IMPLEMENTED

Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
