Test case Operation:     getContainingPGNames
Request Body Parameters:
pTPName: The name of the PhysicalTerminationPoint
Response:
This service returns the names of the Protection Groups containing the given PhysicalTerminationPoint. Note that the response message is handled by getObjectNamesResponse_T.
XML Examples:
none        

Operation:     retrieveSwitchData
Request Body Parameters:
reliableSinkCtpOrGroupName: This is the CTP/FTP that is the output of a service selector in case of the SNC or the group name for which the switch data is being requested. The termination point name must be explicit (a generic endpoint specification may not be used in this case).
Response:
This service is used by the NMS to get the latest switch status on a SNC or a MSP group. It should be noted that although the term MSP was chosen as the original specific protection scheme to which the related behaviour applied was Multiplex Section Protection, the label is now more generally applied to any 1+1 or 1:N Trail protection scheme.
XML Examples:
none        

Operation:     retrieveESwitchData
Request Body Parameters:
ePGPName: This is the equipment protection group name for which the switch data is being requested.
Response:
This service is used by the NMS to get the latest switch status on an equipment protection group. For a retrieval of a revertive M:N group, N ESwitchData_T are returned as a result of retrieveESwitchData (one for each worker equipment instance). For a retrieval of a non-revertive M:N group, N ESwitchData_T are returned as a result of retrieveESwitchData (one for each active equipment instance).
XML Examples:
none        

Operation:     getProtectionGroup
Request Body Parameters:
protectionGroupName: name of the protection group that the client is interested in.
Response:
This operation is used to get the current status of a Protection Group. This service is needed so that even if a filter is established, the NMS can query the status of a protection group.
XML Examples:
none        

Operation:     getEProtectionGroup
Request Body Parameters:
ePGPName: name of the equipment protection group that the client is interested in.
Response:
This operation is used to get the current status of a Equipment Protection Group. This service is needed so that even if a filter is established, the NMS can query the status of a protection group.
XML Examples:
none
