Test case Operation:     getContainingTPs
Request Body Parameters:
tpName: Name of the CTP for which containing CTPs and PTP/FTP are to be reported. The termination point name must be explicit (a generic endpoint specification may not be used in this case). See Object Naming for further detail on FTP naming.
Response:
This service returns a list of the containing TPs given a CTP. This will return an PTP or FTP where there is only one level of containment. In a case of deeper containment this will return a list of CTPs and a PTP or FTP.
XML Examples:
request     response    

Operation:     getContainingTPNames
Request Body Parameters:
Same as getContainingTPs.
Response:
Same as getContainingTPs but returns the name instead of the entire object structures.
XML Examples:
request     response    

Operation:     getContainingSubNetworkNames
Request Body Parameters:
meName: The name of the managed element for which to retrieve the containing subnetwork names.
Response:
This service returns the list of subnetwork names that the Managed Element supplied as an input parameter belongs to.
XML Examples:
request     response    

Operation:     getTp
Request Body Parameters:
tpName: The retrieved TP name.
Response:
This service returns the termination point structure for the given TP name (CTP, FTP or PTP)
XML Examples:
request     response    

Operation:     getManagedElement
Request Body Parameters:
meName: Name of the ME to retrieve
Response:
This service returns the Managed Element for the given managed element name
XML Examples:
request     response    

Operation:     getPotentialFixedCCs
Request Body Parameters:
tpName: Any TP name of the ME. The operation will return the multiplexing or inverse multiplexing scheme in which this TP is involved. The input TP may be either the containing TP or one of the end TPs of the portentialCCList.
Response:
This operation is used to retrieve fixed connection schemes related to normal and inverse multiplexing. See Subnetwork Connection Types for an explanation of fixed SNCs.
Specific Exception: EXCPT_UNABLE_TO_COMPLY - Raised when TP is neither a higher order TP that supports fixed multiplexing nor a lower order TP that supports fixed inverse multiplexing.
XML Examples:
none        

Exceptions EXCPT_NOT_IMPLEMENTED
EXCPT_INTERNAL_ERROR
EXCPT_INVALID_INPUT
EXCPT_ENTITY_NOT_FOUND
EXCPT_COMM_FAILURE
Note: Those exceptions are common to all operations.
Post-conditions In case of success
The requesting OS has all the requested inventory information from the target OS.
In case of failure
The requesting OS does not have the entire requested inventory information from the target OS.
