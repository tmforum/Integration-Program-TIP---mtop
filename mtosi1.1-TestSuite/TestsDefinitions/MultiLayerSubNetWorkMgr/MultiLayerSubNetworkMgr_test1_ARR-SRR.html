Operation:     getTPPool
Request Body Parameters:
tPPoolName: The name of the TP pool or TP for which to retrieve the association.
Response:
This service returns the TPPool structure for the given TPPool name, and the additional attributes numberOfMembers, numberOfIdleMembers, and descriptionOfUse. The attribute containedMembers, i.e. the list of names of contained TPs, can be retrieved by means of the operation getTPGroupingRelationships() with input tpPoolName.
XML Examples:
none        

Operation:     getSNC
Request Body Parameters:
sncName: The name of the SNC.
Response:
This operation will return the SubnetworkConnection structure for the SNC whose name is supplied as a parameter. This operations will provide a bundled SNC structure if the name provided is that of a bundled SNC (see Bundled SNC overview for further details).
XML Examples:
request     response    

Operation:     getAssociatedTP
Request Body Parameters:
tpName: The name of the TP for which to retrieve associated TPs
Response:
This service returns a list of PSR (UPSR or SNCP) associated termination points for the TP whose name is passed as a parameter. Note: The termination point name must be explicit (a generic endpoint specification may not be used in this case). See Object Naming for further detail on FTP naming. When this service is invoked with a TP that is not an edge point, the returned TPs will be on the same Managed Element
XML Examples:
none        

Operation:     getRoute
Request Body Parameters:
sncName: The name of the SNC.
includeHigherOrderCCs: Specifies whether the higher order CCs of other SNCs used to carry the queried SNC have to be included in addition to the CCs of the queried SNC
Response:
This service returns the route for the SNC whose name is specified as a parameter. This operation also works for bundled SNCs.
XML Examples:
none        

Operation:     getRouteAndTopologicalLinks
Request Body Parameters:
sncName: The name of the SNC.
Response:
Like getRoute(), this service returns the route, in terms of crossconnects, for the SNC whose name is specified as a parameter. This service also returns the list of topological links for that SNC that are used in the route.
XML Examples:
none        

Operation:     getSNCsByUserLabel
Request Body Parameters:
userLabel: The userLabel of the SNCs to retrieve.
Response:
This operation will return the SubnetworkConnection structures for the SNCs whose userLabel is supplied as a parameter. It will provide a bundled SNC structure if the name provided is that of a bundled SNC (see Bundled SNC overview for further details). This operation does not use multiple responses, since the number of SNCs returned is expected to be usually 1.
XML Examples:
none
